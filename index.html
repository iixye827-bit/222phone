使用指南

从零开始 (必需)

本应用的代码已托管，您无需关心。此流程仅需您创建一个安全的“钥匙”(PAT)和一个云端“私人数据仓库”，让本应用可以为您安全地存储和同步所有数据。

步骤 1：创建 GitHub 个人访问令牌 (PAT)

这个令牌是应用访问您私人云仓库的专属安全密码。

  1. 访问创建页面：点击此链接直接进入 GitHub Token 创建页面。
  2. 填写基本信息：备注(Note)填写 JellyfishCenterToken，有效期(Expiration)选择 No expiration (永不过期)。
  3. 分配权限 (关键)：只需 勾选 repo 这一项。
  4. 生成并复制：滑动到页面最底部，点击 Generate token 按钮。【⚠️极其重要⚠️】 GitHub 只会显示这一次完整的 Token。请 立即点击复制按钮 将其保存到安全的地方，下一步会马上用到。它应该以 ghp_ 开头。

步骤 2：创建私人数据仓库

此步骤将创建一个独立的、完全私密的仓库，专门用来存放您的所有配置和文件。

  1. 访问创建页面：点击此链接直接进入 GitHub 仓库创建页面。
  2. 填写仓库信息：
    • Repository name (仓库名称)：起一个清晰的名字，例如 My-Jellyfish-Data。
    • 关键：选择 Private (私有)。这是您的私人数据，必须设为私有。
    • 勾选 Add a README file：确保仓库被成功初始化。
  3. 点击 Create repository 按钮完成创建。

步骤 3：在 JellyfishCenter 内完成最终配置

现在，我们将“钥匙”和“数据仓库”信息告诉本应用。

  1. 打开配置面板：长按右下角的 🌞/🌜 按钮展开菜单 → 点击 ☁️ (数据云盘) → 点击标题旁的 ⚙️ (设置)。
  2. 填入信息：
    • 仓库：填写您在 步骤2 创建的**私人数据仓库**地址，格式为 你的GitHub用户名/My-Jellyfish-Data。
    • Personal Access Token：粘贴您在 步骤1 创建并复制好的完整 Token。
  3. 保存并同步：点击 保存配置 按钮。

恭喜！ 您的 JellyfishCenter 现已与云端完全同步，可以开始使用了！

核心操作

编辑标题/副标题: 直接点击页面上的主标题或副标题即可修改。

切换主题: 单击 (短按) 右下角的 🌞/🌜 按钮。

打开功能菜单: 长按 右下角的 🌞/🌜 按钮。

访问链接与返回: 对于外部链接，会在应用内全屏打开。您可以从屏幕**最左侧边缘向右滑动**来关闭当前页面并返回中心。

📱 项目管理

添加新项目：可选择添加`外部链接`或`上传项目`。上传的项目会直接托管在您的GitHub数据仓库中。

管理与删除：在“管理项目”弹窗内，点击`进入管理模式`，即可对现有按钮进行删除。删除“上传项目”类型的按钮会同时删除云端文件。

🎨 主题与个性化

实时调色：可实时调整日间/夜间模式的背景渐变色，以及标题卡片的背景图片。

请注意：按钮、卡片等元素的背景渐变色，其色彩与方向均与当前模式的页面背景相反，以创造深度感。

色彩预设：您可以保存、应用、更新或删除自己喜欢的配色方案，还可以开启“随机主题”功能，在每次启动时随机应用一个已保存的预设。

☁️ 文件云盘

访问与管理：您可以分门别类地管理与“上传项目”相关联的所有文件。建议用来存储备份文件。

云同步状态指示灯：☁️ 图标上的小圆点会实时显示同步状态：

  • 🟢 绿点: 已同步 - 所有本地更改都已成功保存到云端。
  • 🟡 黄点 (闪烁): 同步中 - 正在与云端进行数据交换。
  • 🔴 红点: 同步失败 - 发生错误，请点击图标查看失败原因。
📷 记忆功能

一个集成了备忘录和相册的私人空间，所有数据都安全地存储在您的GitHub私有仓库中。

备忘录

  • 管理: 可以创建、删除文件夹，对笔记进行批量管理。
  • 添加: 在任意文件夹中创建新的笔记。
  • 置顶: 点击笔记右上角的图钉图标 📌，可将其置顶显示。

相册

  • 管理: 创建、删除相册，并可对相册内的照片或照片集进行批量删除。
  • 添加: 上传单张照片或多张照片。
  • 照片集: 当一次性选择多张照片上传时，可以选择将它们创建为一个“照片集”，方便归类。

💫 漫游模式

  • 全屏体验: 点击照片可进入全屏模式，再次点击退出。
  • 轻松切换: 在非全屏模式下，左右滑动可随机切换到下一张照片。
  • 音乐播放器: 底部配有音乐播放器，您可以上传自己的歌曲和歌词，在漫游时享受沉浸式体验。

危险操作：在 GitHub 配置 (⚙️) 弹窗的最下方有“清空所有数据”按钮。此操作会 永久删除 您 GitHub 数据仓库中所有相关数据。此操作不可逆，请极度谨慎。
